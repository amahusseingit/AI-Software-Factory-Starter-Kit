# FAQ

## Is this tied to one tech stack?
No. The core kit is generic. A project spec can define the desired stack.

## Is this only for greenfield projects?
No. It also supports rescue and recovery of incomplete generated systems.

## Can I use this for web, mobile, backend, or integration-heavy systems?
Yes.

## Should I put generic templates in `specs/`?
No. `specs/` is for actual project-specific specifications.

## Can I add domain packs?
Yes, but preferably outside the generic core kit.

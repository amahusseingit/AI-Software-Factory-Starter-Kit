# NFR Pack: Security

## When to activate
Use this pack when the project requires explicit Security expectations.

## Typical concerns
- Project-specific Security goals and constraints
- Measurable acceptance criteria
- Risks if ignored
- Review and release evidence

## Questions to answer
- What does good look like for Security in this project?
- What are the minimum acceptable thresholds or behaviors?
- How will the team verify them?

## Example criteria
- Define measurable targets or explicit expectations for Security
- Add tests, checks, or review evidence that support those expectations
- Prevent release if critical Security obligations remain unverified

## Suggested artifacts
- Acceptance criteria additions
- Test strategy additions
- Review scorecard additions
- Release-readiness evidence

# NFR Pack: Maintainability

## When to activate
Use this pack when the project requires explicit Maintainability expectations.

## Typical concerns
- Project-specific Maintainability goals and constraints
- Measurable acceptance criteria
- Risks if ignored
- Review and release evidence

## Questions to answer
- What does good look like for Maintainability in this project?
- What are the minimum acceptable thresholds or behaviors?
- How will the team verify them?

## Example criteria
- Define measurable targets or explicit expectations for Maintainability
- Add tests, checks, or review evidence that support those expectations
- Prevent release if critical Maintainability obligations remain unverified

## Suggested artifacts
- Acceptance criteria additions
- Test strategy additions
- Review scorecard additions
- Release-readiness evidence

# Security Policy

## Reporting a vulnerability
Please do not open public issues for suspected security vulnerabilities.

Report security concerns privately to the maintainer through the preferred private communication channel for this repository.

## Scope
Security-sensitive areas include:
- guidance related to authentication and authorization
- AppSec testing workflows
- PT readiness and findings handling
- secrets/configuration guidance
- deployment and release-readiness content

## What to include in a report
- affected files or areas
- description of the issue
- possible impact
- suggested remediation if known

# Acceptance Packs

These packs are generic reusable acceptance criteria patterns.

Use them selectively based on your project spec. They are not domain requirements and they should not override the project specification.

Suggested usage:
- `generic-entity-crud-pack.md` for primary entities
- `parent-child-entity-pack.md` for hierarchical data
- `transactional-mutation-integrity-pack.md` for derived-state mutation logic
- `auth-local-jwt-refresh-federated-pack.md` for auth-enabled systems
- `schedule-based-recurring-processing-pack.md` for recurring or scheduled behavior
- `web-ui-state-pack.md` and `mobile-ui-state-pack.md` for UI state completeness

# Contributing

Thanks for contributing to the AI Software Factory Starter Kit.

## Contribution principles
- Keep the core kit **generic**.
- Do not hard-code one business domain into the core repository.
- Prefer reusable skills, templates, references, packs, and workflows.
- Keep changes proportional and traceable.
- Update docs and changelog when adding meaningful capabilities.

## Recommended process
1. Create a branch from the latest main branch.
2. Make focused changes.
3. Update relevant documentation.
4. Add or update examples/templates when useful.
5. Submit a pull request with a clear summary.

## Good contribution areas
- lifecycle improvements
- generic skills and templates
- traceability and evidence patterns
- review gates and scorecards
- rescue and recovery workflows
- AppSec/PT governance
- onboarding and documentation

## Please avoid
- adding project-specific business requirements into the core kit
- broad unrelated cleanup mixed with feature work
- weakening review or evidence expectations

# Spec Quality Checklist

Evaluate the project spec against the following dimensions.

## Business framing
- Clear business goal
- Clear target users or actors
- Explicit success outcomes
- In-scope vs out-of-scope clarified

## Functional behavior
- End-to-end workflows defined
- Alternate flows defined
- Failure flows defined
- Business rules defined
- Validation rules defined

## UI and interaction
- Screen inventory present
- Actions per screen listed
- Navigation transitions clear
- Loading / empty / error / success states addressed
- Role-based visibility defined where relevant

## Data and integration
- Core entities identified
- Relationships and ownership clear
- Integration points identified
- Required request/response expectations identified
- Audit/logging expectations identified when relevant

## Delivery control
- Acceptance criteria exist
- Non-functional requirements addressed where needed
- Open items captured explicitly
- Stack constraints or preferences stated

# V7.3 Usage Guide

## Recommended order

1. Put the project requirements into `specs/<project>-spec.md`.
2. Run `commands/SPEC.md`.
3. Run `commands/PLAN.md`.
4. Run `commands/BUILD.md`.
5. Run `commands/TEST.md`.
6. Run `commands/REVIEW.md`.
7. Run `commands/SHIP.md`.

## New V7.3 controls

### Spec quality analysis
Use the `spec-quality-analysis` skill before planning. The result should be captured in:
- `reviews/spec-gap-report.md`
- `reviews/spec-readiness-scorecard.md`

### Readiness scoring
Use the readiness templates whenever deciding whether to proceed to build, review, rescue, or ship.

### UI workflow and screenshot templates
If the source requirements include screenshots, PDFs, Word documents, Figma exports, or annotated mockups, instantiate the relevant UI templates before planning.

### Review scorecards
Every review should produce at least one scorecard. Use scorecards to support go / conditional go / no-go decisions.

# Review Model

## Required review axes
1. Functional completeness
2. Business rule correctness
3. Stack conformance
4. Architecture consistency
5. UI quality
6. Validation completeness
7. Test adequacy
8. Security basics
9. Cross-platform parity
10. Operational readiness

## Automatic fail conditions
- visible action without handler
- TODO in in-scope flow
- stack drift from SQL Server without approval
- CRUD behavior partially implemented but labeled complete
- Angular or Flutter in-scope module missing parity
- missing validation on create/edit forms
- no evidence file

# Repository Structure

## Core areas
- `commands/` lifecycle interfaces
- `skills/` reusable governed workflows
- `templates/` generic scaffolds
- `references/` guidance, checklists, and taxonomies
- `packs/`, `adr-packs/`, `devops-packs/`, `test-strategy-packs/`
- `specs/` actual project specs only
- `plans/`, `evidence/`, `reviews/`, `adrs/`, `releases/`

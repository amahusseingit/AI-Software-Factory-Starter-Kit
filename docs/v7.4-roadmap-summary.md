# V7.4 Summary

V7.4 adds the next reliability wave on top of V7.3:
- stronger rescue mode
- better traceability automation
- test strategy packs

These upgrades are meant to improve recovery from partially generated systems, make requirement-to-evidence linkage easier to maintain, and improve test planning discipline across different system shapes.

## New capabilities
- rescue gap taxonomy and salvage-vs-rebuild decision guidance
- explicit recovery plan template and rescue scorecard usage
- business rule verification matrix and traceability enhancement guidance
- test strategy packs for common project types and risk profiles
- new skills for traceability governance and test-strategy selection

## Recommended usage
- For weak or incomplete builds, run `commands/RESCUE-PROJECT.md`
- For planning or re-planning, instantiate the traceability templates early
- Before implementation, select a project-level test strategy pack and capture the chosen mix in `plans/`

# V7.4 Usage Guide

## When to use the new V7.4 additions

### Stronger rescue mode
Use when a generated system shows any of the following:
- visible actions with no behavior
- partial CRUD
- broken state refresh
- tests missing or superficial
- stack drift
- web/mobile parity drift
- unclear whether repair or rebuild is the better option

### Traceability automation
Use when you need enterprise-grade linkage between:
- requirements
- acceptance criteria
- business rules
- implementation
- tests
- evidence
- review outcomes

### Test strategy packs
Use before deep implementation, or immediately after rescue assessment, to decide the right test mix for the project.

## Suggested V7.4 flow
1. Place the authoritative project spec under `specs/`
2. Run `commands/START-PROJECT.md`
3. Run `commands/SPEC.md` if gaps remain
4. Instantiate traceability artifacts from `templates/`
5. Select a test strategy pack from `test-strategy-packs/`
6. Run `commands/PLAN.md`
7. Build and test slice by slice
8. If quality collapses midstream, run `commands/RESCUE-PROJECT.md`
9. Run `commands/REVIEW.md`
10. Run `commands/SHIP.md`
